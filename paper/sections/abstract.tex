Return-conditioned sequence policies turn desired return into an inference-time control input. This paper audits a Decision-Transformer-specific failure mode: a larger candidate pool can make a sampled trajectory look more obedient to an ambitious return prompt while making the selected action sequence worse under the real objective. The mechanism is return-support mismatch. When the requested return lies beyond the offline behavior distribution and proxy return underprices risky actions, top-score selection moves mass toward an extrapolative action tail. We introduce a Return-Support Audit with finite-pool accounting, a Return Fantasy Microscope for prompt/support/real-utility gaps, a Tail Phase Diagram, and support-aware repair diagnostics. In the original full run, out-of-support naive selection raises selected proxy return from $9.299$ to $9.468$ while real utility drops from $7.825$ to $7.381$ as $N$ grows from $1$ to $64$. The expanded suite extends the stress to $N=256$: selected proxy return rises by $0.224$, selected real utility falls by $0.629$, and selected risk rises by $0.0071$. Pilot real-utility calibration improves $N=256$ real utility by $1.146$ over raw selection; a behavior-likelihood filter improves it by $1.216$; the oracle diagnostic gap is $1.239$. On a CPU-light CartPole-v1 benchmark tier, raw high-$N$ proxy selection raises the short-horizon proxy from $0.887$ to $0.962$ while true episode return falls from $309.1$ to $45.9$; support filtering and behavior cloning recover the capped return of $500.0$. The result is a bounded audit protocol for deciding when high return prompts reflect utility and when they are extrapolation warnings, not a claim of D4RL, MuJoCo, Atari, or deployment-scale dominance.
